Проверка конфигурации

Настройка, подготовка  и запуск проверки конфигурации выполняются из формы "Запуск проверки". Форма вызывается или из панели инструментов или из пункта главного меню "Запуск проверки" из подменю "Проверка".

На этапе "Сведения о проверяемой конфигурации" указываются сведения о конфигурации, которую нужно проверить, а именно: - указывается проверяемая конфигурации; при необходимости вводится новый элемент в справочнике "Конфигурации" и в нем указываются сведения о проверяемой конфигурации;

    • устанавливается режим "Проверять расположенную в каталоге версию"
    • при необходимости корректируется путь к демонстрационной базе проверяемой конфигурации, пользователь для этой базы с полными правами и его пароль.

После нажатия на кнопку "Далее" определяется структура конфигурации и выполняется тестирование получения каждой формы и Каждого макета конфигурации. При попытке получить форму или макет конфигурации может возникнуть ошибка выполнения. В этом случае необходимо продолжить тест, нажав на кнопку "Продолжить тест". Формы и макета, при попытке получить которые возникли ошибки, фиксируются и при продолжении тестирования и сборе сведений о конфигурации они будут пропущены.

После завершения тестирования форм и макетов для всех объектов конфигурации будут назначены ответственные за объекты конфигурации из последней проверяемой версии и выполнен переход к следующему этапу "Настройка проверки". По результат назначения ответственных в верхней части формы будет выведена итоговая информация. Корректировка ответственных за объекты конфигурации  выполняется в отдельной форме, вызов которой выполняется по кнопке "Скорректировать".

Корректировка ответственных выполняется непосредственно для объекта конфигурации в выведенном дереве, либо ответственные могут быть назначены по сведениям, полученным из файла Excel, где в первой колонке указывается тип объекта, во второй – имя объекта, в третьей – ответственный за объект.

После назначения ответственных, необходимо отметить в списке программы проверки, которые требуется выполнить для конфигурации. Выбор в списке программы проверки откроет её форму, где можно проинспектировать или скорректировать её состав.

После завершения настроек запускается проверка конфигурации по кнопке "Запуск".
После этого в автоматическом режиме выполняется сбор сведений о конфигурации и её проверка. В зависимости от размера конфигурации и длительность сбора сведений и проверки может быть от нескольких минут для простой конфигурации до 4 – 6 часов для больших конфигурации, например УПП, редакции 1.2.

После завершения проверки будет открыт отчет по проверке конфигурации, работа с которым подробно описана в сценарии "Получение отчета по проверке и распределение недочетов".

 

Проверка версии конфигурации, по которой данные собраны ранее

При обновлении или уточнении правил и программ проверки полезно сразу перейти к проверке, без повторного сбора сведений о конфигурации.
Такая вариант проверки выполняется в форме "Запуск проверки". Форма вызывается или из панели инструментов или из пункта главного меню "Запуск проверки" из подменю "Проверка".

На этапе "Сведения о проверяемой конфигурации" указываются сведения о версии конфигурации, которую нужно проверить, а именно:

    • указывается конфигурация, версию которой необходимо проверить;
    • устанавливается режим "Проверять версию, по которой данные были собраны ранее";
    • в поле "Версия" указывается версия, которую необходимо проверить.

После нажатия на кнопку "Далее"  выполняется переход к следующему этапу -  "Настройка проверки". По текущему состоянию назначенных ответственных в верхней части формы будет выведена итоговая информация. Корректировка ответственных за объекты конфигурации  выполняется в отдельной форме, вызов которой выполняется по кнопке "Скорректировать". Подробнее корректировка ответственных описана выше.

После назначения ответственных, необходимо отметить в списке программы проверки, которые требуется выполнить для конфигурации. После завершения проверки будет открыт отчет по проверке конфигурации, работа с которым подробно описана в сценарии "Получение отчета по проверке и распределение недочетов".
