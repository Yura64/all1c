Форма предназначена для редактирования информации о проверяемой конфигурации. На данной форме необходимо заполнить следующие поля:

·         «Наименование» – поле, в котором указывается краткое название конфигурации.

·         «Полное наименование» – поле, в котором указывается полное название конфигурации.

·         «Ответственный» – поле, в котором указывается ответственный за конфигурацию.

·         «Версия» – поле, содержащее номер сборки проверяемой конфигурации.

·         «от» – поле, в котором указывается дата публикации релиза проверяемой конфигурации.

·         «Включать в отчетность» – флажок, который указывает на необходимость включать сведения о проверяемой конфигурации в отчетность.

·         «Выполнять рассылку результатов проверки» – флажок, который указывает на необходимость рассылки отчетов о результатах проверки конфигурации по электронной почте ответственным.

Чтобы проверить подключение к указанной базе по COM-соединению необходимо нажать на кнопку «Проверить подключение к базе».

Также необходимо заполнить следующие вкладки:

Вкладка «Настройки доступа»
«Платформа» – группа параметров, включающая сведения о проверяемой конфигурации:

·         «Версия» – поле, содержащее версию платформы, на которой запускается проверяемая конфигурация.

·         «Запуск» – путь к файлу платформы, на которой запускается проверяемая конфигурация.

 

«Параметры подключения к демобазе» – группа параметров, включающая сведения о демонстрационной базе:

·         «С демобазой» – путь к демобазе проверяемой конфигурации;

·         «Пользователь» – имя пользователя демобазы с полными правами;

·         «Пароль» – пароль пользователя с полными правами.

 

«Параметры подключения к хранилищу» – группа параметров, включающая сведения о хранилище информационной базы:

·         «Хранилище» – путь к хранилищу информационной базы;

·         «Пользователь» – имя пользователя хранилища информационной базы;

·         «Пароль» – пароль пользователя.

 

Вкладка «Проверяемые требования»
На данной вкладке устанавливается состав требований, предъявляемых к конфигурации:

·         «Система стандартов» – правила проверки в соответствии со стандартами и методиками разработки конфигураций.

·         «Орфография» – правила проверки орфографии слов конфигурации.

·         «1С:Совместимо (1С:Предприятие 8.3)» – правила соответствия с требованиями разработки «1С:Совместимо» (выбрано по умолчанию).


Чтобы выбрать необходимые требования и группы требований, нужно отметить их флажками. Также можно указать вариант проверки конфигурации, выбрав в поле «Вариант проверки» один из следующих вариантов:

·         «Полная проверка» – выполнение проверки на требования групп «Система стандартов» и «Орфография».

·         «1С:Совместимо» – выполнение проверки на требования группы «1С:Совместимо».

Имеется возможность добавить собственный вариант проверки, указав в списке состав требований и нажав на кнопку «Сохранить вариант». При необходимости пользовательские варианты проверки можно удалить, нажав на кнопку «Удалить вариант», однако предопределенные варианты удалять нельзя.
При установке флажка «Расширенная проверка по дополнительным требованиям» при выполнении проверки будут учитываться внутренние стандарты 1С.
При установке флажка «Проверять объекты с префиксом удалить» будут проверяться объекты конфигурации с префиксом «Удалить».

«Библиотека» – переключатель, при выборе которого можно проверить конфигурацию на любые требования.
«Конфигурация» – переключатель, при выборе которого требования, входящие в подгруппу «Разработка и использование библиотек», будут исключены из проверки вне зависимости от того, выбраны они или нет.

 

Вкладка «Исключения из проверки»
На этой вкладке представлен список объектов, который будет исключен при тестировании проверяемой конфигурации. В колонке «Вид исключения» необходимо выбрать один из видов исключений:

·         «Включать с префиксом» – будут проверяться только объекты, имя которых начинается с префикса, указанного в колонке «Значение».

·         «Исключить с префиксом» – будут проверяться все объекты, исключая объекты, имя которых начинается с префикса, указанного в колонке «Значение».

·         «Включать подсистему» – будут проверяться только объекты, входящие в подсистему, имя которой указано в колонке «Значение».

·         «Исключать подсистему» – будут проверяться все объекты, исключая объекты, входящие в подсистему, имя которой указано в колонке «Значение».

·         «Включать тип объектов» – будут проверяться только объекты, тип которых совпадает со значением, указанным в колонке «Значение».

·         «Исключать тип объектов» – будут проверяться все объекты, исключая объекты, тип которых совпадает со значением, указанным в колонке «Значение».

·         «Включать объект» – будет проверяться только объект, полный путь к которому совпадает со значением, указанным в колонке «Значение».

·         «Исключать объект» – будут проверяться все объекты, исключая объект, полный путь к которому совпадает со значением, указанным в колонке «Значение».

При установке флажка «Запоминать исключения без привязки к месту обнаружения» будет выполняться поиск ошибок в особенностях без учета места их обнаружения.

 

Вкладка «Библиотечные конфигурации»
На этой вкладке представлен список конфигураций, которые указываются в качестве библиотечных. При проверке конфигурации учитываются особенности библиотечных конфигураций.

 

Вкладка «Особенности»
На этой вкладке представлен список особенностей конфигурации.
Гиперссылка «Показать особенности конфигурации» формирует отчет «Особенности».
При включенном флаге «Выгружать особенности по расписанию в файл» будет выполняться автоматическая выгрузка особенностей по заданному расписанию в файл (xml, zip).
Настройки для выгрузки по расписанию:

·         «Открыть расписание» – гиперссылка, которая вызывает окно настройки расписания.

·         «Следующее время выгрузки» – поле, в котором отображается следующее по расписанию время выгрузки особенностей.

·         «Пользователь» – пользователь, под которым запустится выгрузка особенностей по расписанию.

·         «Варианты выгрузки»:

o   «Полная» – будут выгружены все особенности конфигурации вместе с авторами особенностей;

o   «Без персональных данных» – будут выгружены все особенности конфигурации без авторов особенностей.

Выгрузить особенности также можно вручную. Для этого необходимо нажать на кнопку «Выгрузить», в открывшемся окне указать каталог и файл, в который необходимо выгрузить особенности, и нажать на кнопку «Сохранить».
Кнопка «Выгрузить без персональных данных» выполняет действие аналогичное кнопке "Выгрузить", но без выгрузки авторов особенностей.
Загрузка особенностей выполняется только вручную. Для этого необходимо нажать на кнопку «Загрузить», в открывшемся окне указать путь к файлу выгруженных особенностей и нажать на кнопку «Открыть».
После этого список особенностей данной конфигурации будет дополнен списком особенностей из загружаемого файла.
Чтобы очистить весь список особенностей, нужно нажать на кнопку «Очистить».

Вкладка «Причины особенностей»
На данной вкладке представлен список причин особенностей конфигурации. При добавлении новой причины особенности ее можно выбрать из справочника причин особенностей. При нажатии на кнопку «Заполнить из особенностей» список заполнится на основании тех особенностей, которые уже есть в отчете «Особенности» для данной конфигурации.

 

Вкладка «Расписание»
На данной вкладке можно запланировать запуск проверки конфигурации по определенному расписанию.
Здесь имеются следующие поля:

·         «Проверять конфигурацию по расписанию» – флажок, при установке которого проверка конфигурации будет осуществляться по заданному расписанию.

·         «Открыть расписание» – гиперссылка, которая вызывает окно настройки расписания.

·         «Следующее время запуска» – поле, в котором отображается следующее по расписанию время запуска проверки конфигурации.

·         «Регистрировать все ошибки как особенности» – флажок, при установке которого в процессе выполнения проверки все ошибки конфигурации будут отмечены как особенности.

·         «Пользователь» – пользователь, под которым запустится проверка конфигурации по расписанию.

·         «Вариант проверки» – вариант проверки конфигурации; может принимать следующие значения:

o   «Текущие настройки» – выполнение проверки на требования, указанные в карточке конфигурации на вкладке «Проверяемые требования».

o   «Полная проверка» – выполнение проверки на требования групп «Система стандартов» и «Орфография».

o   «1С:Совместимо» – выполнение проверки на требования группы «1С:Совместимо».

·         «Шаблон письма» – текст, который будет автоматически добавлен в письмо при рассылке всем ответственным за ошибки.

 

Вкладка «Расширения»
На данной вкладке представлен список расширений конфигурации.
Имеет три сценария проверки:

·         «Конфигурация (без расширений)» - будет проверяться только конфигурация. 

·         «Только расширения» -  будут проверяться расширения, для которых установлен флаг в списке расширений.

·         «Конфигурация и расширения» - будут проверяться конфигурация и расширения, для которых установлен флаг в списке расширений.

Кнопка «Обновить» - актуализирует список расширений конфигурации.
