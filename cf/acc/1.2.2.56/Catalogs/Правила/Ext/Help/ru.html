В справочнике содержатся правила, по которым проверяются конфигурации. Правила проверки могут быть скорректированы пользователем. Правила проверки могут быть приведены к составу и содержанию, предусмотренных по умолчанию.

Настройка программ и правил проверки начинается с корректировки состава и содержания правил. Для этого необходимо открыть форму списка справочника «Правила». Форма вызывается из пункта главного меню «Правила» в подменю «Настройка».

Все правила расположены по группам. Определять состав групп и принадлежность правил какой либо группе рекомендуется по функциональному назначению, то есть по тому что правило и группа правил будут проверять.

Для редактирования содержания правила требуется открыть форму элемента. Для настройки правила требуется выполнить несколько действий: 1. Определить типы объектов конфигурации, которые будут проверятся правилом, например:  «Форма», «Общая форма». Выполняется это в табличном поле «Типы объектов для проверки». По нажатию на кнопку «Добавить все»  добавляются все типы объектов. Это облегчает определение нужных типов объектов в случае, если требуется проверять правило для объектов всех типов или почти всех, за исключением некоторых.

Алгоритм проверки объекта конфигурации указывается в поле «Алгоритм проверки». Он пишется на встроенном языке 1С:Предприятие. Для доступа к сведениям о данных объекта используются вызовы предопределенных функций, например: «ПолучитьСвойства()».

Алгоритм проверки правила может быть дополнен выполнением проверки с помощью отдельно разработанной обработки. Такая обработка называются в рамках конфигурации дополнением и хранится в справочнике «Дополнения». Если для правила используется дополнение, то оно указывается в поле «Дополнение», а запуск дополнения выполняется в алгоритме проверки вызовом процедуры «ВыполнитьДополнение()».

Правило в большинстве случае проверяет выполнение определенного пункта стандарта для проверяемого объекта. Стандарт указывается в поле "Стандарт". Работу правила можно предварительно проверить. Для этого нужно открыть форму проверки правила по кнопке «Проверить».  В этой форме необходимо указать версию конфигурации, на которой требуется выполнить проверку, а также конкретный объект конфигурации, если требуется выполнить только его проверку.

Проверка работы правила выполняться по кнопку «Проверить». Результаты проверки выводятся не в журнал проверки, а в табличное поле в этой же форме. После окончания испытаний форму нужно закрыть.

После завершения корректировки состава и содержания правил корректируются программы проверки. Это выполняется в форме списка справочника «Программы проверки», которая вызывается из пункта «Программы проверки» подменю «Настройка» главного меню. Программы проверки выводятся при настройке проверки в соответствии с иерархией, заданной в данном справочнике. Корректировка программы проверки выполняется в форме элемента. Для программы проверки необходимо указать лаконичное и понятное название, составить описание программы проверки: её назначение и особенности и после этого определить состав. Для определения состава необходимо выбрать те правила проверки в поле «Все правила», которые будут составлять данную программу. При выборе правило появляется в поле «Состав». Для исключения правила из состава программы проверки необходимо выбрать его в поле «Состав».

Алгоритмы, записанные в правилах проверки используют встроенный язык 1С:Предприятие совместно с процедура и функциями, определенными в модуле обработки "Ядро".

Процедуры и функции работы со свойствами метаданных
ПолучитьСвойства()
Функция возвращает структуру со свойствами объекта проверяемой конфигурации. В колонке "Имя" содержатся имена свойств, в колонке "Значение" - значения.


ПолучитьСвойстваРодителя()
Функция возвращает структуру со свойствами родителя объекта проверяемой конфигурации. В колонке "Имя" содержатся имена свойств, в колонке "Значение" - значения.


ПолучитьТип()
Возвращает тип текущего объекта конфигурации, если такой для него определен.


ПолучитьПодсистемы()
Возвращает список подсистем, к которым относится текущий объект конфигурации


ПолучитьПрава()
Возвращает права роли, которой является текущий объект конфигурации


Процедуры и функции работы с формами
ПолучитьСвойстваФормы()
Функция возвращает структуру со свойствами формы. Если проверяемый объект не является формой, возвращается "Неопределено".


ПолучитьЭлементыФормы()
Функция возвращает массив со строками дерева значений, которые отображают в дереве формы элементы формы.


ПолучитьСвойстваЭлементаФормы( ЭлементФормы)
Функция возвращает структуру со свойствами переданного элемента формы.


Процедуры и функции работы с макетами
ПолучитьСвойстваМакета()
Функция возвращает структуру со свойствами макета. Если проверяемый объект не является макетом, возвращается "Неопределено".


ПолучитьСодержаниеМакета()
Функция возвращает дерево значений с содержанием макета.


Процедуры и функции работы с модулями
ПолучитьСоставМодулей()
Возвращает состав модулей объекта конфигурации.


ПолучитьТекстМодуля(ИмяМодуля)
Возвращает текст переданного модуля объекта конфигурации.


ПолучитьСтруктуруМодуля(ИмяМодуля)
Возвращает структуру переданного модуля объекта конфигурации.


Процедуры и функции работы со справкой
ПолучитьТекстСправочнойИнформации()
Возвращает текст справки объекта конфигурации.


Процедуры и функции проверки орфографии
ОчиститьОшибкиОрфографии()
Очищает все ошибки, обнаруженные при проверки орфографии.


ПроверитьОрфографию(ТекстДляПроверки, ОбъектПроверки , Дополнительно)
Проверяем переданный текст (ТекстДляПроверки) на орфографические ошибки
Совместно с текстом могут быть переданы дополнительные
реквизиты, характеризующие место обнаруженной ошибки:
 - объект проверки (ОбъектПроверки),
 - прочие дополнительные сведения (Дополнительно).


ПолучитьОшибкиОрфографии()
Возвращает таблицу с обнаруженными ошибками орфографии с момента последней очистки ошибок орфографии.


Прочие процедуры и функции
ЗапуститьДополнение()
Запускает дополнение.


Зарегистрировать(Комментарий, Критичность, ДополнительныеСведения)
Регистрирует найденный недочет:

 - суть замечаний (Комментарий),
 - оценка критичности обнаруженной проблемы (0-9) (Критичность)
 - произвольные дополнительные сведения (ДополнительныеСведения).
