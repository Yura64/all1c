Форма записи регистра сведений "Найденные ошибки".

С помощью указания кода ошибки в тексте объекта конфигурации можно исключить этот объект из проверки.

Исключение записывается следующим образом:
- для конфигурации на русском языке (свойство конфигурации "Основной язык" = "Русский"): // АПК:<код_ошибки><-выкл|вкл> <описание_причины>
- для конфигурации на английском языке (свойство конфигурации "Основной язык" = "Английский"): // ACC:<код_ошибки><-off|on> <описание_причины>

Примеры:

Исключение в тексте модуля:
1. В той же строке модуля, для которой необходимо отключить проверку:

2. Для блока модуля:

Исключение в свойстве "Комментарий" объекта конфигурации:
3. Для всего объекта конфигурации - в комментарии этого объекта:

4. Для подсистемы и ее объектов - в комментарии подсистемы:

Можно при проверке пропустить объекты подсистемы, но проверить какой-нибудь конкретный объект этой подсистемы. Для этого необходимо в комментарии подсистемы написать: "АПК:566-выкл не проверяем синхронные вызовы". А в комментарии к объекту оставить запись: "АПК:566-вкл проверяем синхронные вызовы.".
