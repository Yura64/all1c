Для разработки технически качественных решений на платформе 1С:Предприятие 8 необходимо придерживаться выработанных стандартов  и рекомендаций, выпускаемые как "Система стандартов и методик разработки конфигураций для платформы 1С:Предприятие 8" (размещена на сайте http://its.1c.ru/, а также на диске ИТС).

Кроме этого, техническое качество основывается на ряде очевидных соображений общего характера, как, например, следование правилам орфографии в текстах конфигурации, контроль обращения к прикладным объектам, отсутствующим в конфигурации  и т. д.

Выполнение проверок  и исправление найденных недочетов на постоянной основе приводит к выработке у разработчиков навыков правильного кодирования и проектирования и, как следствие, повышает качество будущих работ, значительно снижает число допущенных разработчиком недочетов.

Выполнение проверки в ручном режиме требует значительных усилий и хорошего знания "Системы стандартов и методик разработки конфигураций для платформы 1С:Предприятие 8".  Даже для конфигураций малого объема выполнение таких проверок на постоянной основе проблематично.

Конфигурация "Автоматизированная проверка конфигураций" выполняет работы по проверке технического качества конфигураций в автоматическом режиме:

    • выявляет недочеты;
    • содержит базу стандартов и методик разработки, прикрепляет к найденным недочетам текст стандарта;
    • позволяет гибко настраивать проверку;
    • распределяет недочеты по ответственным;
    • учитывает при проверке особенности конфигурации, указанные ответственным;
    • выводит подробные результаты проверки в виде табличного документа с заданной группировкой.
Выполнение проверки, хранение и обработка результатов проверки выполняется в одной информационной базе для нескольких конфигураций.

Проверка конфигурации производится на основании сведений, полученных из файлов, выгруженных в режиме «Конфигуратор», и сведений, полученных в режиме «1С:Предприятие». Для этого производится сбор сведений и подготовка данных для проверки. Из демонстрационной базы проверяемой конфигурации без участия пользователя получают сведения о метаданных, тексты модулей и информацию о формах, макетах и справке. После этого выполняется проверка конфигурации и обработка полученных результатов.

Проверка конфигурации заключается в проверке выполнения заранее настроенных правил для каждого объекта конфигурации и формирования записей о всех найденных в процессе проверки недочетах в журнале проверки.

Правила проверки хранятся в справочнике. В правиле определен алгоритм проверки на встроенном языке и типы объектов конфигурации, которые проверяются с помощью этого правила. Данные о конфигурации для работы алгоритма проверки правила доступны через объекты сведений – специально реализованные обработки для получения и анализа сведений о проверяемой конфигурации.

Для объединения правил в группу по целевому назначению проверок (например, «Проверка орфографии» и «Проверка форм») используются программы проверки. Они задаются в справочнике "Требования", а состав правил в программе определяется через регистр сведений, так как одно и то же правило может потребоваться проверять для разных программ. Программы и правила можно экспортировать и импортировать, а в соответствии с системой стандартов для 1С:Предприятие 8 используется базовое заполнение этих справочников.

Перед началом проверки пользователь устанавливает какие необходимо выполнить программы проверки. Из состава этих программ будет определен список правил для проверки конфигурации.

Проверка заключается в последовательной проверке каждого объекта конфигурации.

Для проверки объекта по его типу определяют набор правил проверки и последовательно проверяют каждое правило.  Сводные результаты проверки  - отчет по проверке - формируются после проверки всех объектов конфигурации в виде дерева с группировкой по различным критериям.

Для организации работы над исправлением недочетов нескольких разработчиков предусмотрено назначение ответственных за недочеты. Ответственных можно назначить либо непосредственно в отчете по проверке, либо перед выполнением проверки назначить ответственного за конкретные объекты конфигурации.
